InnerClasses ClassFile

Document all inner classes and static nested class of this class, including recursive ones.

Mandatory & at most one iff. this class have inner classes.

 A length-prefixed list where each item consists of:
  1. Reference to a CONSTANT_Class_info (the inner class)
  2. Reference to a CONSTANT_Class_info (the direct container class of the inner class)
  3. Reference to a CONSTANT_Utf8_info (simple name of the inner class)
  4. u2 access big flags
EnclosingMethod ClassFile

Mandatory & at most one iff. this is a local class or anonymous class.
  1. Reference to a CONSTANT_Class_info (the direct container class of this class)
  2. Reference to a CONSTANT_NameAndType_info (the method in which this local class is declared)
SourceDebugExtension ClassFile

Stores extra debugging information.

JVMS only requires that the debugging information must be text-based and stored in the modified UTF-8 encoding, and does not impose further semantics on this attribute. Other specification may impose structure on the stored information. e.g. JSR 45, the spec that proposed this attributes, defines the Source Map (SMAP) format that is stored within this attribute. SMAP is used by the JSP technologies to store mapping between JSP source and generated servlet class.

An arbitrary byte array.
BootstrapMethods ClassFile Represent subroutines used to produce dynamically-computed constants and dynamically-computed call sites. A length-prefixed list where each item consists of:
  1. Reference to a CONSTANT_MethodHandle_info (the callable stuff)
  2. A list of references to loadable entries in the constant pool. (The argument to use)
Module (since 9) ClassFile

This attribute is only allowed in a module-info.class, and is written by the javac

This attribute encodes all information presented in a module-info.java source, including the declared moudle name, all directives, etc.

  1. Reference to CONSTANT_Module_info (the current module)
  2. A u2 bit flag module_flags indicating certain properties of this module.
  3. Reference to a CONSTANT_Utf8_info (the module version)
  4. 5 Length-prefixed lists, one for each of requires, exports, opens, use, provides directives
ModulePackages (since 9) ClassFile

This attribute is only allowed in a module-info.class

Indicates a few packages within this module.

JVMS allows this attributes to indicate any packages in this module. JVMS recommends that this attribute indicates the exports-ed, opens-ed, provides with-ed packages.

In current practice this attribute is always created by the jar program during packaging, and is never created by javac; and jar always does this by simply copying the exports-ed, opens-ed, provides with-ed packages from the Module attribute created by javac

A list of references to CONSTANT_Package_info.
ModuleMainClass (since 9) ClassFile

This attribute is only allowed in a module-info.class

Indicate the main class of the module. java -m <MODULE_NAME> will respect this attribute as program entrance.

In current practice the only officially supported way of creating this attribute is by calling jar with --main-class option during packaging.

Reference to a CONSTANT_Class_info
NestHost (since 11) ClassFile

Indicate the nest host for the current class or interface.

Facilitate the nest based access control of Java 11.

 Reference to a CONSTANT_Class_info (the nest host for the current class or interface)
NestMembers (since 11) ClassFile A list of references to CONSTANT_Class_info (member of the nest hosted by the current class)
Record (since 16) ClassFile

Indicate that this is a Java 16 record class (instead of depending on bits in ClassFile.access_flags), and document the name and type of the record's components (i.e. fields).

A list of record_component_info, one for each component in the record, defined as:
  1. Reference to a CONSTANT_Utf8_info (unqualified name of the component)
  2. Reference to a CONSTANT_Utf8_info (field descriptor of the component)
  3. A list of attribute_info specific to this component.

Code method_info

Method execution information, including the bytecode instructions you are looking for.
  1. u2 max_stack: the maximum depth of the operand stack of this method
  2. u2 max_locals: the number of local variables in the local variable array allocated upon invocation of this method
  3. A list of bytes: the bytecode instruction
  4. A list of exception handler
  5. A list of attribute_info specific to this Code_attribute instance.
Exceptions method_info Document the throws declaration of this method. A list of references to CONSTANT_Class_info (one for each throws-ed class )
RuntimeVisibleParameterAnnotations method_info Document @Retention(RetentionPolicy.RUNTIME) annotations tagged on the parameters of the method. A list of parameter_annotations, one for each parameter of the method. Each parameter_annotations is a list of annotations structure.
RuntimeInvisibleParameterAnnotations method_info Document @Retention(RetentionPolicy.CLASS) annotations tagged on the parameters of the method.
AnnotationDefault method_info

Only allowed when this class file represents an annotation definition, where each method_info represents an annotation element (i.e. those method-like members)

Identify the default value of this annotation element.

An element_value structure.
MethodParameters method_info Document the formal parameters of a method (i.e. excluding the explicit this parameter) A list of item, one for each formal parameter of this method, defined as:
  1. Reference to a CONSTANT_Utf8_info (the parameter name)
  2. access_flags bit flag: whether this parameter is final, synthetic, or mandated.
LineNumberTable method_info -> Code_attribute

Mapping between positions in the code array (bytecode instructions) and line numbers of the original source code.

This attribute is extensively used by debuggers, but is optional for production class release.

 A list of item, each defined as:
  1. a u2 number: an index into the code array, starting from which comipiled instructions are from a new line in the source code.
  2. a u2 number: the source line number.

This attribute is extensively used by debuggers to determine the value of a given local variable during the execution of a method; but is optional for production class release. Multiple occurrences of this attributes in a same parent are allowed in any order.

A list of items, each defined as:
