option name semantics possible values
core.eol Determine the character sequence git uses as line separator when
  1. creating files in the working directory by checking out an existing commit
  • lf
  • crlf
  • native (default)
core.autocrlf Known as the end-of-line conversion, this option controls
  1. whether to convert an LF to CRLF when checking out a commit into the working directory.
  2. whether to convert a CRLF to LF when making a commit into the repository.
  • true: convert both 1 and 2
  • false (default): don't convert
  • input: convert only 1
(false is not mentioned in the doc, but supported)
